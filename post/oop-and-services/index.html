Eric Evan’s Domain driven design

Service, when an operation does not conceptually belong to any object. Following the natural contours of the problem, you can implement these operations in services. See also Service (systems architecture).

Martin Fowler

The Service Layer, defines an application’s boundary with a layer of services that establishes a set of available operations and coordinates the application’s response in each operation.
